根据您的指示，已将 **PDF 处理相关功能** 的状态更新为 **Unimplemented (暂未实现)**。这意味着在当前交付版本中，接口将仅支持图像输入，PDF 相关的参数值和逻辑已被标记为不可用。

以下是更新后的完整 API 需求交付文档：

### 📋 接口需求文档：PP-OCRv5 通用文字识别 (v1.1 - PDF Unimplemented)

#### 1\. 接口概览 (Interface Overview)

| 项目 | 内容 | 备注 |
| :--- | :--- | :--- |
| **接口名称** | PP-OCRv5 通用文字识别 | 百度 AI Studio 服务化部署 |
| **请求协议** | HTTP / HTTPS | |
| **请求方法** | `POST` | |
| **请求路径** | `/ocr` | 需配合控制台分配的 Base URL 使用 |
| **数据格式** | `application/json` | |
| **鉴权方式** | Token 认证 | Header 中携带访问令牌 |

-----

#### 2\. 请求头定义 (Request Headers)

| 参数名 | 值示例 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `Content-Type` | `application/json` | **是** | 指定负载格式 |
| `Authorization`| `token <Access_Token>` | **是** | **注意格式**：Token 值前需加 "token " 前缀（含空格） |

-----

#### 3\. 请求体参数 (Request Body)

**数据结构**：JSON Object

| 参数名 | 类型 | 必填 | 默认值 | 说明与业务逻辑 |
| :--- | :--- | :--- | :--- | :--- |
| **`file`** | String | **是** | - | **输入文件数据**。<br>1. **Base64**：**图像**内容的 Base64 编码字符串。<br>2. **URL**：服务器可访问的**图像**链接。<br>\<span style="color:red"\>**[Unimplemented]** 暂不支持 PDF 文件流或 PDF URL。\</span\> |
| `fileType` | Integer | 否 | `1` | **文件类型**。<br>`1`: 图像文件 (Supported)<br>\<span style="color:grey"\>`0`: PDF 文件 \<span style="color:red"\>**(Unimplemented)**\</span\>\</span\><br> |
| `useDocOrientationClassify` | Boolean | 否 | `false` | **文档方向矫正**。<br>是否识别并矫正 0°/90°/180°/270° 旋转的图片。 |
| `useDocUnwarping` | Boolean | 否 | `false` | **图片扭曲矫正**。<br>是否矫正弯曲、褶皱或倾斜的文档图片。 |
| `useTextlineOrientation` | Boolean | 否 | `false` | **文本行方向矫正**。<br>针对具体文本行进行 0°/180° 翻转判断。 |
| `textDetLimitSideLen` | Integer | 否 | `64` | **图像长边限制**。<br>图像预处理时的长边限制值，值越大显存占用越高。 |
| `textDetThresh` | Number | 否 | `0.3` | **检测像素阈值**。<br>像素点得分大于该值才判定为文字像素。 |
| `textDetBoxThresh` | Number | 否 | `0.6` | **检测框阈值**。<br>检测框内平均得分大于该值才判定为文字区域。 |
| `textDetUnclipRatio` | Number | 否 | `1.5` | **检测扩张系数**。<br>控制检测框的大小，值越大框越宽。 |
| `textRecScoreThresh` | Number | 否 | `0.0` | **识别置信度阈值**。<br>识别结果得分低于该值的文本将被过滤丢弃。 |
| `visualize` | Boolean | 否 | `false` | **开启可视化**。<br>`true`: 返回结果中包含带检测框的图片 URL（增加耗时）。 |

-----

#### 4\. 响应体结构 (Response Body)

**成功响应 (HTTP 200)**

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `logId` | String | 请求唯一标识 (UUID)，用于排查问题。 |
| `errorCode` | Integer | 状态码，`0` 表示成功。 |
| `errorMsg` | String | 状态信息，成功时为 `"Success"`。 |
| `result` | Object | 具体的业务数据对象。 |
| └ `ocrResults` | Array | 识别结果列表。 |
|   └ `prunedResult` | String | **核心字段**：识别出的文本内容。 |
|   └ `ocrImage` | String | (当 `visualize=true` 时) 可视化结果图的 URL。 |
|   └ *(其他)* | - | 可能包含坐标 `points` 或置信度 `score` (视具体模型输出而定)。 |

**失败响应**

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `errorCode` | Integer | 非 `0` 值 (如 503, 504 等)。 |
| `errorMsg` | String | 具体的错误描述信息。 |

-----

#### 5\. 状态码参考 (Status Codes)

  * `200`: 成功 (Success) - **需进一步判断 body 中的 errorCode**
  * `429`: 请求过多 (Too Many Requests) - 达到每日配额上限
  * `503`: 服务不可用 (Service Unavailable) - 资源紧张
  * `504`: 网关超时 (Gateway Timeout) - 处理时间过长
